# B-e2e: 端到端检索策略对比实验报告

> **实验日期**: 2026-06-29
> **实验类型**: 不同检索策略注入 system prompt 后的端到端回答质量对比
> **评测路线**: 端到端 (LLM 回答 + 关键词命中)
> **模型**: Gemini (gemini-flash-lite-latest)

---

## 一、实验目的

回答核心问题：**不同记忆检索策略，对最终 LLM 回答质量的影响差多少？**

这与之前的"有记忆 vs 无记忆"（A/B 实验）不同——这次**所有策略都有记忆**，区别在于**怎么检索和注入**。

---

## 二、实验设计

### 2.1 四种检索策略

| 策略 | 怎么注入 system prompt | 代表什么 |
|---|---|---|
| **Noesis** | 经代理，走完整检索 + 置信度门控 + token 预算 (1200) | 本系统 |
| **朴素RAG** | 取该用户全部非tentative记忆塞入 system | 最强召回基线 |
| **最近窗口** | 只取最近 5 条记忆塞入 system | ChatGPT 默认 |
| **无记忆** | 空 system | 对照下界 |

### 2.2 测试集

- 3 个用户画像（张伟/李雷/韩梅梅），每个 5 道题，共 **15 题**
- 每题用 4 种策略各问一次 = **60 次 LLM 调用**
- 评分：关键词命中（客观、可复现）

---

## 三、实验结果

### 3.1 总览

| 排名 | 策略 | 命中率 | vs 无记忆 |
|---|---|---|---|
| 🥇 | **朴素RAG** | **14/15 (93%)** | +40pp |
| 🥈 | Noesis | 11/15 (73%) | +20pp |
| 🥈 | 最近窗口 | 11/15 (73%) | +20pp |
| 4 | 无记忆 | 8/15 (53%) | — (基线) |

### 3.2 反直觉发现：朴素RAG 在小规模下优于 Noesis ⚠️

**这是一个必须诚实报告的重要发现。** Noesis (73%) 低于朴素RAG (93%)，有 3 题是 Noesis 答错而 RAG 答对。

#### 根因分析

| 问题 | Noesis 回答 | 朴素RAG 回答 | 根因 |
|---|---|---|---|
| 我之前用什么做向量检索? | ❌ Milvus (错) | ✅ sqlite-vec (对) | Noesis top-k **没召回** sqlite-vec 那条 |
| 我在学什么编程语言? | ❌ "没有记录" | ✅ Rust (对) | Noesis top-k **没召回** Rust 那条 |
| 帮我推荐一个数据库 | ❌ 推荐了 Milvus | ✅ PostgreSQL | Noesis top-k **没召回** PostgreSQL 偏好 |

**核心原因**: Noesis 只注入 top-k（受 1200 token 预算限制），偶尔会漏掉相关信息；朴素RAG 把**全部记忆**塞进去，所以绝不会漏。

### 3.3 这个发现的真实含义（关键洞察）

朴素RAG 召回率高，是因为它**牺牲了 token 效率换全量注入**。但：

| 画像 | 全量记忆数 | 全量字符数 | 约 token 数 |
|---|---|---|---|
| zhang_wei | 25 条 | 370 字符 | ~190 |
| li_lei | 14 条 | 207 字符 | ~105 |
| han_meimei | 21 条 | 329 字符 | ~170 |

**问题**: 当前记忆池太小（14-25 条），全量注入也才几百 token，**朴素RAG 的"贵"根本没体现**。Noesis 的 token 效率优势在这个规模下无从发挥。

#### 推论（需后续实验验证）

当记忆规模增长到 100-500 条时：
- **朴素RAG** 的全量注入会爆炸（几千-几万 token），成本飙升、可能超 context 上限
- **Noesis** 仍稳定在 1200 token 预算，靠精准检索维持质量

**这正是"记忆规模退化曲线"实验要验证的——朴素RAG 会在大规模下退化，而 Noesis 不会。**

### 3.4 Noesis vs 最近窗口 vs 无记忆

排除朴素RAG 后，Noesis 的优势清晰：

| 策略 | 命中率 | 说明 |
|---|---|---|
| **Noesis** | 73% | 检索相关记忆注入 |
| 最近窗口 | 73% | 只注入最近 5 条（侥幸持平）|
| 无记忆 | 53% | 完全不注入 |

Noesis 和最近窗口在当前规模持平（都 73%），但**机制完全不同**——Noesis 靠检索，窗口靠运气（恰好最近的有相关的）。大规模下窗口会退化，Noesis 不会。

---

## 四、结论

1. **在记忆规模小（<30 条）时，朴素RAG 的全量注入因召回全面而占优**，Noesis 的 token 效率优势未体现。
2. **Noesis (73%) 显著优于无记忆 (53%)**，证明记忆注入本身有效。
3. **当前实验的最大局限是记忆规模太小**，无法体现 Noesis 在大规模下的核心优势（token 效率）。
4. **强烈建议下一步做"记忆规模退化曲线"实验**——让记忆从 10 涨到 500，观察各策略的命中率 + token 消耗曲线。预计朴素RAG 会因 token 爆炸而退化，Noesis 保持稳定。

---

## 五、实验局限（诚实报告）

| 局限 | 影响 | 改进 |
|---|---|---|
| **记忆规模太小 (14-25条)** | 朴素RAG 全量注入的优势被放大，Noesis 效率优势未体现 | 扩展到 100-500 条 |
| **未测 token 消耗** | 无法量化 Noesis 的效率优势 | 加入 token 计数指标 |
| **Noesis top-k 偶尔漏召回** | 小规模下召回不如全量 | 调大 token 预算或改进检索 |
| **关键词命中粗糙** | "微服务"这类热门词无记忆也能命中 | 用更冷门的偏好 |
| **3 个画像 15 题** | 统计显著性不足 | 扩大到 ≥50 题 |
